# SRL地理探究智能体 - 教师端改造说明文档

## 一、改造概述

根据需求文档，本次改造将原有的师生双端系统简化为**仅保留教师端**，并通过外部API获取学生对话数据，使用DIFY平台进行元认知分析。

## 二、主要修改内容

### 2.1 首页改造 (`src/app/page.tsx`)

**修改前：**
- 包含学生端和教师端两个入口卡片
- 学生端可以进行探究对话

**修改后：**
- 移除学生端入口
- 仅保留教师端入口
- 简化页面布局，突出教师端功能

**修改原因：**
根据需求，系统只需要保留教师端功能，学生对话通过外部系统完成。

---

### 2.2 新增学生对话API (`src/app/api/conversations/route.ts`)

**新增功能：**
- 从外部API `http://api-vpc.aix101.com/api/vpc/english/summary` 获取学生对话数据
- 过滤 `big_comment` 为 `"SRL-test"` 的记录
- 按学生名称分组，构建对话列表
- 将原始数据转换为前端需要的格式

**数据处理逻辑：**
```typescript
1. 调用外部API获取所有对话记录
2. 过滤出 big_comment === 'SRL-test' 的记录
3. 按 student_name 分组
4. 将 answer（学生发言）和 comment（AI回复）转换为消息列表
5. 按时间排序（API返回的是最新在前，需要反转）
6. 返回格式化的对话列表
```

**API响应格式：**
```json
{
  "success": true,
  "conversations": [
    {
      "id": "conv-{studentName}-{timestamp}",
      "studentName": "学生姓名",
      "messages": [
        {
          "id": "{messageId}-answer",
          "role": "user",
          "content": "学生发言内容",
          "timestamp": 1234567890
        },
        {
          "id": "{messageId}-comment",
          "role": "assistant",
          "content": "AI回复内容",
          "timestamp": 1234567891
        }
      ],
      "startTime": 1234567890,
      "lastUpdateTime": 1234567899,
      "status": "completed"
    }
  ],
  "total": 10
}
```

## 三、技术架构

### 3.1 数据流

```
外部学生对话API (api-vpc.aix101.com)
        ↓
   /api/conversations (数据获取与格式化)
        ↓
   教师端页面 (展示对话列表)
        ↓
   用户选择对话 → 点击分析
        ↓
   /api/analyze (调用DIFY API)
        ↓
   DIFY平台 (https://dify.aix101.com/v1)
        ↓
   返回分析结果 → 展示报告
```

### 3.2 API依赖

| API | 用途 | 地址 |
|-----|------|------|
| 学生对话API | 获取学生对话数据 | http://api-vpc.aix101.com/api/vpc/english/summary |
| DIFY API | 元认知分析 | https://dify.aix101.com/v1 |

## 四、使用说明

### 4.1 教师使用流程

1. **访问首页**
   - 打开系统首页
   - 点击"进入教师端"按钮

2. **查看对话列表**
   - 系统自动从外部API加载学生对话
   - 显示学生姓名、消息数、对话时长等信息
   - 可点击"刷新数据"手动更新

3. **查看对话详情**
   - 点击任意对话卡片进入详情页
   - 查看学生与AI的完整对话记录
   - 查看对话统计信息

4. **生成分析报告**
   - 在对话详情页点击"开始分析"
   - 系统调用DIFY API进行元认知分析
   - 分析完成后自动跳转到报告页面

5. **查看分析报告**
   - 元认知编码分布（PL/MO/RE/EV）
   - SRL阶段表现（计划/监控/调节）
   - 评价维度得分
   - 综合评语和教学建议
   - 详细编码结果

### 4.2 数据说明

**学生对话数据来源：**
- 学生通过外部系统与AI进行探究对话
- 对话记录存储在外部数据库
- 教师端通过API获取 `big_comment` 为 `"SRL-test"` 的记录

**分析结果说明：**
- PL（计划与定位）：学生对目标的设定、任务的拆解
- MO（监控与察觉）：学生对认知状态的审视
- RE（策略调节）：学生主动改变探究路径
- EV（评价与总结）：对成果和策略的审定

## 六、注意事项

### 6.1 网络要求
- 需要能够访问 `api-vpc.aix101.com`（学生对话API）
- 需要能够访问 `dify.aix101.com`（DIFY分析API）

### 6.2 数据格式
- 外部API返回的 `answer` 字段为学生发言
- 外部API返回的 `comment` 字段为AI回复
- 只有 `big_comment` 为 `"SRL-test"` 的记录会被展示

### 6.3 错误处理
- API请求失败时会显示错误提示
- 用户可点击"重试"按钮重新加载
- 分析失败时会返回默认结果并提示重试
